# 시나리오 1 : 가능한 DDoS 공격

A-1. 잠재적인 데이터 출처는 무엇입니까? 
1) 트래픽 상승 현상 확인(SNMP 그래프 등) 후 조직의 내부 IP 대역으로 들어오기 위해 거치는 라우터, 방화벽, 보안 장비 등의 로그에서 원인이 되는 출발지 IP 및 트래픽 원인 확인
2) 내부 대역에서 운영되는 웹 서버의 웹 로그에서 비정상적으로 많은 요청 트래픽이 존재하는지 확인

A-2. 도움이 되는 정보를 포함할 가능성이 가장 높은 잠재적인 데이터 출처와 그 이유는 무엇입니까?
1) 방화벽 및 라우터의 세션 데이터, 내부 대역에서 실서비스중인 웹 서버의 웹 로그
2) 외부 네트워크 대역에서 내부로 들어오는 모든 트래픽이 해당 장비들을 거치기 때문에… 
3) 그리고 해당 장비들에는 트래픽의 세션 정보(출발지-목적지 IP, 송수신 바이트수, etc.)들이 남아있어서… 
4) 해당 웹 서버의 웹 로그에는 비정상적으로 많은 요청 트래픽이 존재하거나, Slow DDoS와 같은 공격 요청 트래픽이 존재할 가능성이 존재해서…

A-3. 어떤 데이터 소스가 가장 먼저 검사될 것이며 그 이유는 무엇입니까?
1) 방화벽 및 라우터의 세션 데이터
2) 네트워크의 대역폭이 고갈된 정황이 있으므로, 해당 네트워크로 향하는 트래픽이 모두 통과하게 되는 라우터 및 방화벽을 검사하는 것이 먼저라고 판단…

A-4. 어떤 포렌식 도구와 기술이 가장 많이 사용될 것입니까? 어떤 다른 도구와 기법을 사용할 수 있습니까?
1) 네트워크 패킷 수집 및 분석 > 와이어샤크, 엑셀, Tcp dump, etc.
2) 방화벽 및 라우터 세션 수집 및 분석 > 네트워크 장비 세션 수집 스크립트, 로그 수집기, 엑셀, etc.
3) 웹 로그 수집 및 분석 > Web Log Expert, 엑셀, etc.

A-5. 조직 내의 어떤 그룹과 개인이 포렌식 활동에 참여할 것입니까?
1) ISP > 대용량 대역폭 고갈성 공격에 대한 정보 제공 
2) 네트워크 담당자 > 네트워크 트래픽 수집 자료를 제공, 실시간 네트워크 상황 모니터링 및 요청사항에 대한 응답
3) 사고대응 전문가 > 가용성 고갈성 공격에 대한 대응
4) 포렌식 전문가 > 네트워크 장비 상에 남아 있는 대용량 대역폭 고갈성 공격에 대한 흔적 조사 및 유발지 역학 조사(DDoS 봇넷에 의한 공격인지? 단일 공격자에 의한 공격인지?), 공격의 목적 및 공격자의 의도 파악 등

A-6. 외부 당사자와 어떤 커뮤니케이션이 있을 수 있습니까?
1) ISP에 대한 해당 공격에 대한 트래픽 정보 요청
2) 네트워크 담당자에 대한 네트워크 트래픽 수집 자료 요청 및 실시간 네트워크 상황 모니터링 및 알림 요청
3) 사고대응 전문가에 대한 실시간 공격에 대한 내부 네트워크 대역으로의 트래픽 유입 차단 요청 및 기타 공격 유입 여부 파악 요청

A-7. 포렌식 관점에서, 시나리오가 다른 날이나 다른 시간에 발생했다면(정규 시간대 근무외 시간) 다르게 수행되는 것은 무엇입니까?
1) 추후 정규 시간대에 분석을 위해 필요한 정보들을 수집 및 보관(네트워크 트래픽 정보, 로그 등)
2) 긴급한 건에 대해서는 해당 사고대응 담당자에 의한 신속한 차단 및 가용성 유지를 위한 조치가 필요
3) 정규 시간 및 근무외 시간 대에 발생한 공격은 신속하게 관련 담당자에게 현재 상황 및 정황을 사실(Fact)을 기반으로 정확하게 설명
4) 그리고 관련 담당자가 어떤 행동(Action)을 취할 것인지에 대한 확실한 응답을 받고, 그 행동(Action)에 필요한 자신의 행동(Action)을 응답 받는 것이 필요

A-8. 포렌식 관점에서, 시나리오가 다른 물리적 위치(현장과 현장외)에서 발생했다면 어떻게 다르게 수행됩니까?
1) 필요한 조치에 대해 해당 사이트 담당자에게 전달 
2) 추후 유발지 역학 조사를 위해 필요한 정보들에 대한 수집 및 보관을 요청 
3) 현재 상황에 대한 자세한 정보를 요청 후, 필요한 조치에 대해 응답

추가 A-1. DDoS 공격이 다른 주에 있는 네트워크에서 오는 것처럼 보이는 경우, 포렌식 활동이 어떻게 변경될까요? 아니면 다른 나라에서이면?
1) 로컬 ISP에서 현재까지 수집 가능한 정보를 요청하고 해당 DDoS 대역에 대한 로컬 ISP단의 대응을 요청
2) 리모트 ISP에 대해 공조를 요청하고, 해당 DDoS 대역에 대한 대응을 요청 

추가 A-2. DDoS 공격이 비즈니스 파트너의 네트워크에서 오는 것처럼 보이는 경우 포렌식 활동이 어떻게 변경됩니까?
1) 로컬 네트워크에서 수집한 트래픽 정보를 해당 파트너 담담자에게 보낸 후, 원인 파악 및 대응을 요청



# 시나리오 2 : 온라인 지불 문제

A-1. 잠재적인 데이터 출처는 무엇입니까?
1) 해당 조직의 온라인 청구서 웹 앱과 관련된 웹 서버 오류 로그 
2) 해당 조직의 온라인 청구서 웹 서버와 연결된 데이터베이스 서버의 오류 로그
3) 해당 데이터베이스 서버의 상태 로그
4) 온라인 청구서 웹 앱의 사용자 UI 혹은 요청 방법 등의 변경 이력 확인(비기술적 문제일 경우 대비)

A-2. 도움이 되는 정보를 포함할 가능성이 가장 높은 잠재적인 데이터 출처와 그 이유는 무엇입니까?
1) 해당 온라인 청구서 웹 앱의 개발 담당자 인터뷰(사용자 인터페이스 변경이나, 결제 방식의 변경 등 개발적 이슈에 대한 확인을 위함)
2) 웹 서버의 오류 로그
3) 데이터베이스의 오류 로그 및 상태 로그
4) 온라인 청구서 웹 앱의 소스코드

A-3. 어떤 데이터 소스가 가장 먼저 검사될 것이며 그 이유는 무엇입니까?
1) 온라인 청구서 웹 앱의 개발 이슈
2) 보통 웹 앱의 사용성 문제는 기술적인 문제라고 생각하기 쉽지만, IT를 잘모르는 일반적인 사용자들은 기존의 웹 앱 UI에서 조그마한 변화만 생겨도 이를 인지하지 못하고 기존의 방법대로 하기를 원하며, 추가적인 시도 없이 시스템의 문제로 생각하여 컴플레인을 제기하기 때문에...

A-4. 어떤 포렌식 도구와 기술이 가장 많이 사용될 것입니까? 어떤 다른 도구와 기법을 사용할 수 있습니까?
1) 웹 서버의 오류 로그 > 엑셀, 오류 로그 파서, etc.
2) 데이터베이스의 오류 로그 엑셀, 오류 로그 파서, etc.
3) 웹 로그(해당 웹 서버에 대한 슬로우로리스(Slowroris)류의 공격 의심) > 엑셀, 웹 로그 파서, etc.

A-5. 조직 내의 어떤 그룹과 개인이 포렌식 활동에 참여할 것입니까?
1) 웹 개발 담당자 > 해당 웹 앱의 개발 이슈 인터뷰 
2) 웹 서버 담당자 > 웹 서버 로그 수집 및 요청 응답
3) 데이터베이스 서버 담당자 > 데이터베이스 서버 로그 수집 및 요청 응답 

A-6. 외부 당사자와 어떤 커뮤니케이션이 있을 수 있습니까?
1) 기존에는 잘 돌아가던 시스템인지 확인 요청
2) 기존과 비교해서 시스템 상에서 최근에 어떤 변경 사항이 있었는지 확인 요청

A-7. 포렌식 관점에서, 시나리오가 다른 날이나 다른 시간에 발생했다면(정규 시간대 근무외 시간) 다르게 수행되는 것은 무엇입니까?
1) 추후 정규 시간대에 분석을 위해 필요한 정보들을 수집 및 보관(관련 로그 등)
2) 그리고 관련 담당자가 어떤 행동(Action)을 취할 것인지에 대한 확실한 응답을 받고, 그 행동(Action)에 필요한 자신의 행동(Action)을 응답 받는 것이 필요

A-8. 포렌식 관점에서, 시나리오가 다른 물리적 위치(현장과 현장외)에서 발생했다면 다르게 수행됩니까?
1) 필요한 조치에 대해 해당 사이트 담당자에게 전달 
2) 추후 유발지 역학 조사를 위해 필요한 정보들에 대한 수집 및 보관을 요청 
3) 현재 상황에 대한 자세한 정보를 요청 후, 필요한 조치에 대해 응답

추가 A-1. 문제는 새로운 사용자에 대한 명확한 지침이 없는 등의 기술적이지 않은 것이 원인이 될 수 있습니다. 조사의 기술적 측면과 비 기술적 측면은 어떻게 조율되고 균형을 이루어야 합니까?
1) 최근의 업무적인 개발 이슈 및 시스템의 변경사항에 확인하고, 그런 이력이 존재하지 않으면 외부의 공격으로 판단 가능

추가 A-2. 개인정보보호 고려사항은 포렌식 도구 및 기법의 사용에 어떤 영향을 미칩니까?
1) 데이터베이스 데이터 조사의 경우, 개인정보와 밀접하기 때문에 수집 및 분석 후 파기 

추가 A-3. 애플리케이션 개발자가 운영상의 문제로 인해 문제가 발생했다는 확신이 들면 포렌식 도구와 기법을 어떻게 사용합니까?



# 시나리오 3 : 알 수 없는 무선 액세스 지점

A-1. 잠재적인 데이터 출처는 무엇입니까?
1) 발견된 허가되지 않은 무선 액세스 지점 상의 로그
2) 해당 액세스 지점에 접속한 단말 상의 로그
3) 해당 액세스 지점이 공용망으로 나가면서 거치는 네트워크 장비 상의 로그

A-2. 도움이 되는 정보를 포함할 가능성이 가장 높은 잠재적인 데이터 출처와 그 이유는 무엇입니까?
1) 발견된 허가되지 않은 무선 액세스 지점 상의 로그
2) 문제의 원인이 되는 장비이기 때문에 해당 장비를 조사하면 추가적인 출처를 얻을 가능성이 존재…
